# 기능 명세: 더보기 및 프로필 (More & Profile)

사용자의 프로필 정보 수정, 앱 설정 관리 및 기타 부가 정보를 제공하는 기능입니다.

## 1. 개요 (Overview)
- **목적:** 등록된 아기 정보를 관리하고 앱의 부가 기능을 설정함.
- **주요 가치:** 정보 유연성, 사용자 맞춤형 환경 제어.

## 2. 사용자 시나리오 (User Scenarios)
1. 사용자는 하단 탭의 '더보기' 메뉴를 선택한다.
2. 현재 등록된 아기의 닉네임과 정보를 확인한다.
3. 사용자는 '편집' 버튼을 눌러 아기 정보를 수정할 수 있는 편집 모드로 진입한다.
4. 수정된 정보는 '저장' 버튼을 눌러 즉시 반영한다.
5. 앱 버전 정보 및 기타 정책 문서를 열람할 수 있다.

## 3. 기능적 요구사항 (Functional Requirements)
- **[F1] 프로필 조회:** 현재 등록된 아기 정보(닉네임, 성별 등)를 화면에 노출한다.
- **[F2] 정보 수정:** 등록된 아기 정보(닉네임, 성별 등)를 수정하고 로컬 DB에 저장한다.
- **[F3] 앱 관리:** 버전 정보 노출 및 약관/정보 페이지 이동 링크를 제공한다.
- **[F4] UI 상태 관리 (MVI):** 읽기 모드와 편집 모드를 전환하여 정보를 관리한다.

## 4. 데이터 모델 (Data Model)
- `BabyInfo` 모델을 공유하여 사용함 (온보딩 기능과 동일).

## 5. 현재 구현 상태 (Implementation Status)
- **[완료] 레포지토리 연동:** `BabyRepository`의 조회 및 수정 로직 연동 완료.
- **[완료] UI (MVI):**
    - `MoreContract.kt`: 상태(State)와 편집 인텐트(Intent) 정의.
    - `MoreViewModel.kt`: 프로필 정보 로드 및 업데이트 비즈니스 로직.
    - `MoreScreen.kt`: 단일 화면 내에서 조회 모드와 편집 모드를 제공하는 UI 구현 완료.
- **[진행 중] 부가 메뉴 구현:** `SCREEN_FLOWS.md`에 정의된 '백업', '알림 설정', '초음파 앨범' 등 상세 메뉴는 고도화 단계에서 추가 구현 예정.
